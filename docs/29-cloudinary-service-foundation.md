This document closes Phase 5.3 and defines the server-side Cloudinary service foundation for TRP Booking.

## Goal

Phase 5.3 adds the minimum server-side Cloudinary code foundation after Phase 5.2 validated the required Cloudinary environment variables.

The goal is intentionally limited:

```text
Add the Cloudinary SDK dependency.
Configure Cloudinary server-side using validated environment variables.
Add helper functions for accommodation image folders and deterministic public IDs.
Add a delivery URL helper for future public rendering.
Do not upload images yet.
Do not add admin image management UI yet.
Do not write image records to the database yet.
Do not add booking, payment, email, calendar, iCal, or PMS features.
```

## Official Cloudinary Basis

Cloudinary documents the Node.js SDK as the official Node.js integration library and recommends using the `v2` SDK style.

Cloudinary also documents that required configuration values can be set with `cloud_name`, `api_key`, and `api_secret`, and that applications must not expose the API secret.

This subphase follows the explicit variable approach selected during Phase 5.1 instead of introducing a single `CLOUDINARY_URL` variable.

## Server-Side Cloudinary Client

The Cloudinary client foundation lives in:

```text
lib/cloudinary/client.ts
```

It uses:

```text
getCloudinaryEnv()
CLOUDINARY_CLOUD_NAME
CLOUDINARY_API_KEY
CLOUDINARY_API_SECRET
```

The client helper configures the SDK with `secure: true` and returns the configured Cloudinary v2 client.

The helper intentionally throws if executed in a browser context because Cloudinary provider credentials must remain server-side only.

## Folder Helpers

Folder and public ID helpers live in:

```text
lib/cloudinary/folders.ts
```

Supported future folder pattern:

```text
trp-booking/{environment}/accommodations/{propertySlug}
```

Supported future public ID pattern:

```text
trp-booking/{environment}/accommodations/{propertySlug}/{sortOrder}-{imagePurpose}
```

Example:

```text
trp-booking/dev/accommodations/bungalow-refugio-perfecto/01-cover
```

## Delivery URL Helper

The delivery helper lives in:

```text
lib/cloudinary/delivery.ts
```

It can generate secure Cloudinary image URLs from an existing Cloudinary public ID.

It does not:

```text
Upload files
Delete assets
Rename assets
Persist records
Call database writes
Expose secrets to the browser
```
