# 23 — Auth.js Configuration

This document closes Phase 4.3 and defines the initial Auth.js / NextAuth configuration for TRP Booking.

## Phase

```text
Phase: Phase 4 — Admin Authentication Foundation
Subphase completed by this document: 4.3 Auth.js configuration

```

## Goal

Phase 4.3 installs and configures Auth.js for the private admin authentication foundation without exposing admin UI or adding database-backed Auth.js adapter tables.

The configured strategy is the one selected in Phase 4.1:

```text
Auth.js / NextAuth
Google OAuth provider
JWT session strategy
Server-side admin allowlist
No Prisma adapter initially
No credentials/password login initially
```

next-auth 5.0.0-beta.31

## Auth Configuration

The root `auth.ts` file exports the standard Auth.js helpers:

```text
handlers
auth
signIn
signOut
```

The configuration uses:

```text
providers: Google OAuth
session.strategy: jwt
secret: AUTH_SECRET
trustHost: AUTH_TRUST_HOST
```

Google OAuth credentials are read only from validated server-side environment variables:

```text
AUTH_GOOGLE_ID
AUTH_GOOGLE_SECRET
```

## Auth Route Handler

The Auth.js route handler was added at:

```text
app/api/auth/[...nextauth]/route.ts
```

Supported methods:

```text
GET
POST
```

The route is configured for the Node.js runtime and forced dynamic behavior because authentication must not be statically optimized.

## Admin Allowlist Behavior

Admin access is still controlled server-side through:

```text
AUTH_ALLOWED_ADMIN_EMAILS
```

During Google sign-in, the Auth.js `signIn` callback requires:

```text
Provider must be google
Google profile email must be verified
Email must be present
Email must match the normalized server-side allowlist
```

Emails are normalized with:

```text
trim
lowercase
```

No real admin emails are hardcoded in code or documentation.

## JWT Session Behavior

The session strategy remains JWT.

For allowlisted users, the JWT/session callbacks safely expose only:

```text
user.email
user.role = ADMIN
```

The session does not expose:

```text
AUTH_SECRET
AUTH_GOOGLE_SECRET
provider access tokens
raw OAuth responses
database connection strings
```

## TypeScript Session Shape

The project now augments Auth.js types in:

```text
types/next-auth.d.ts
```

The session user can include:

```text
role?: "ADMIN"
```

This prepares the project for Phase 4.4 route protection without adding admin UI yet.
